  animation-fill-mode 这个属性是动画执行前和执行后指定采用的样式。

  执行前 可能有delay的延时，所以一个动画的完整步骤是

  [ 延时阶段，可能为0s  ] -> [ 动画执行阶段 ] -> [ 执行完毕之后]

  1，如果 animation-fill-mode 设置为 none，表示延时阶段 不采用第一帧的样式，执行完毕之后不采用最后一帧的样式。  执行完动画，整个样式恢复为默认

  2，如果 animation-fill-mode 设置为 backwards，表示延时阶段 采用第一帧的样式，也就是0% 或者from的样式，如果延时设置为0，实际上是是感受不到的。 这时候没有设置执行完毕后的样式，所以，执行完毕后，整个样式恢复为默认

  3，如果 animation-fill-mode 设置为 forwards, 表示执行完毕之后，采用最后一帧的样式，也就是100% or from的样式，动画执行完毕之后使用最后的是最后的样式。

  4，如果 animation-fill-mode 设置为 both，表示动画执行前的延时阶段（可能为0），采用第一帧，执行完毕之后采用最后一帧的样式。

  
所以一般情况下，设置为none 和 backwards没啥作用
